# Auth

Google OAuth 2.0 + JWT session management. Sprint 1 implementation — unchanged since.

---

## Flow

```
1. Frontend → POST /graphql { InitiateAuth }
2. Controller → redirect to Google OAuth
3. User authenticates with Google
4. Google → GET /auth/callback?code=...
5. Controller exchanges code for Google profile
6. Issue JWT (access + refresh tokens) → set httpOnly cookie
7. Frontend → POST /graphql (with JWT)
8. AuthMiddleware validates JWT → inject user into context
```

---

## JWT Tokens

| Token | TTL | Storage |
|-------|-----|---------|
| Access token | Short (minutes) | httpOnly cookie |
| Refresh token | Long (days) | httpOnly cookie + Redis |

- **Issuer:** `appmeta.ControllerIssuer`
- **Secret:** `JWT_SECRET` env var
- Refresh tokens stored in Redis for revocation

---

## Enrollment JWT (separate from user JWTs)

Connector enrollment uses a different JWT issued by the controller on behalf of an authenticated admin:

```go
// Generated by: graph/resolvers/connector.resolvers.go → GenerateEnrollmentToken mutation
// Verified by:  internal/connector/token.go → VerifyEnrollmentToken()
// JTI burned:   internal/connector/token.go → BurnEnrollmentJTI() (Redis atomic GET+DEL)
```

Claims: `connector_id`, `workspace_id`, `trust_domain`, `jti` (single-use ID)
