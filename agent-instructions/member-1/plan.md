## Phase-by-Phase Plan

### Phase 1 — GraphQL Operation Files (can start Day 1)

Create the `.graphql` files. These do NOT depend on the backend schema update landing yet — you write them against the planned schema so they're ready when codegen runs.

**File: `admin/src/graphql/connector-mutations.graphql`**

```graphql
mutation CreateRemoteNetwork($name: String!, $location: NetworkLocation!) {
  createRemoteNetwork(name: $name, location: $location) {
    id name location status createdAt
  }
}

mutation DeleteRemoteNetwork($id: ID!) {
  deleteRemoteNetwork(id: $id)
}

mutation GenerateConnectorToken($remoteNetworkId: ID!, $connectorName: String!) {
  generateConnectorToken(remoteNetworkId: $remoteNetworkId, connectorName: $connectorName) {
    connectorId
    installCommand
  }
}

mutation RevokeConnector($id: ID!) {
  revokeConnector(id: $id)
}

mutation DeleteConnector($id: ID!) {
  deleteConnector(id: $id)
}
```

**File: `admin/src/graphql/connector-queries.graphql`**

```graphql
query GetRemoteNetworks {
  remoteNetworks {
    id name location status createdAt
    connectors { id name status lastSeenAt version hostname }
  }
}

query GetConnectors($remoteNetworkId: ID!) {
  connectors(remoteNetworkId: $remoteNetworkId) {
    id name status lastSeenAt version hostname publicIp certNotAfter createdAt
  }
}
```

### Phase 2 — Page Scaffolds and Routing (can start Day 1)

Build the page components with loading states and layout. Wire up routes. You can use mock/placeholder data until codegen output is available.

**Route additions in `App.tsx`:**

```
/remote-networks              → RemoteNetworks.tsx
/remote-networks/:id/connectors → Connectors.tsx
```

Both are protected routes inside `ProtectedLayout`.

**Sidebar addition:** Add "Remote Networks" link between Dashboard and Settings in `Sidebar.tsx`.

### Phase 3 — RemoteNetworks Page

- "Add Network" button opens an inline modal with name input + location dropdown
- Location options: `HOME`, `OFFICE`, `AWS`, `GCP`, `AZURE`, `OTHER`
- Calls `createRemoteNetwork` mutation on submit
- Displays network cards: name, location badge, active connector count, "View Connectors" link
- Delete button visible only when a network has zero connectors
- Calls `deleteRemoteNetwork` mutation on delete
- Uses `GetRemoteNetworks` query with `cache-and-network` fetch policy

### Phase 4 — Connectors Page

- URL: `/remote-networks/:id/connectors`
- Breadcrumb: Remote Networks → [Network Name]
- "Add Connector" button opens `InstallCommandModal`
- Table columns: name, status badge, last seen (relative time), hostname, version, actions
- Status badge colors: `PENDING`=gray, `ACTIVE`=green, `DISCONNECTED`=amber, `REVOKED`=red
- Actions: Revoke button (calls `revokeConnector`), Delete button (calls `deleteConnector`)
- **Polls `GetConnectors` every 30 seconds** — use Apollo `pollInterval: 30000`
- No manual refresh button needed

### Phase 5 — InstallCommandModal

Two-step flow:

1. **Step 1:** Connector name input → calls `generateConnectorToken` mutation
2. **Step 2:** Shows monospace code block with the install command + one-click copy button + warning banner: _"This token expires in 24 hours and works only once."_

The `installCommand` string comes from the mutation response. Display it exactly as returned — do not construct it on the frontend.

### Phase 6 — Codegen and Wiring

**Blocker:** You need Member 4's `schema.graphqls` update merged before codegen will succeed.

Once the schema is merged:

```bash
cd admin && npm run codegen
```

This regenerates `admin/src/generated/*`. Replace placeholder/mock data in your pages with the real generated hooks.

## Special Instructions

1. **SPIFFE is invisible to the frontend.** You will never see SPIFFE IDs, trust domains, or certificate details in the UI. Don't add any fields for them.

2. **The `installCommand` is opaque.** Display it as a pre-formatted string from the API. Never parse or construct it client-side.

3. **Polling, not WebSockets.** Use Apollo `pollInterval` for connector status updates. 30 seconds matches the heartbeat interval.

4. **Do not modify the Apollo client setup.** The existing `authLink` and `errorLink` already handle JWT auth and refresh. Your new queries/mutations go through the same pipeline automatically.

5. **Generated files are gitignored build artifacts.** Never hand-edit files in `admin/src/generated/`. Run `npm run codegen` to regenerate them.

6. **Follow existing page patterns.** Look at `Dashboard.tsx` and `Settings.tsx` for query loading, skeleton placeholders, and error handling patterns. Match the existing style.
